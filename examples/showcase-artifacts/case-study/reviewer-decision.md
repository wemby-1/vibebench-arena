# Reviewer Decision

The reviewer should use VibeBench artifacts as decision support, not as automatic approval.

## Possible Decisions

Merge:

The change is scoped, checks pass, risk findings are understood, and the reviewer agrees with the implementation.

Request changes:

The evidence packet shows missing checks, unclear risk, incomplete documentation, or implementation details that need more work.

Reject:

The change is too broad, risky, poorly justified, or not aligned with the original request.

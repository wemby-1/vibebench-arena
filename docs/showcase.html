AI coding is fast. Review evidence is scattered.

A vibe-coded patch can look plausible before anyone has checked the diff, risk surface, command output, release readiness, or artifacts. Vibe-coding needs reproducible checks, local artifacts, and shareable proof that humans can review before trusting the change.

The problem

AI agents can produce code quickly, but the evidence around the change often lives across terminals, chat notes, CI logs, and memory.

The response

VibeBench collects the local quality trail into commands, summaries, packets, manifests, comparisons, and release audit records.

The product loop

VibeBench is built around the moment after an agent changes files and before a person decides what to trust, merge, or release.

Change

Start with an AI-assisted diff from Codex or another coding agent.

Check

Run configured checks, gates, dry-runs, and readiness commands.

Explain

Turn outcomes into readable summaries and review context.

Package

Bundle artifacts, manifests, reports, and release audit records.

Prove

Create proof packets with HTML, JSON, Markdown, and checksums.

Review

Inspect evidence locally or from the GitHub Actions artifact.

Evaluate it in 5 minutes

Run the demo, inspect JSON, generate a proof packet, and check strict readiness without depending on a hosted service.

  • python3 -m vibebench demo
  • python3 -m vibebench demo --json
  • python3 -m vibebench proof --output-dir .vibebench/proof-packet --zip
  • python3 -m vibebench doctor --strict

Who can use it

VibeBench is useful when teams want AI-assisted coding to leave a review trail, not just a patch.

Solo developers

Builders using Codex or AI coding agents can run a local quality gate before committing.

Reviewing teams

Teams reviewing AI-assisted pull requests can ask for evidence packets and CI artifacts.

Maintainers

Maintainers can compare generated work against checks, summaries, and release readiness.

AI tooling builders

Tool builders can use local quality gates and proof artifacts around agent workflows.

Honest limits

VibeBench is intentionally careful about what the current project proves.

  • It is not a replacement for SWE-bench.
  • It is not a hosted enterprise product yet.
  • It does not guarantee correctness, funding, stars, or adoption.
  • It is a local-first quality-control layer for AI-assisted coding workflows.

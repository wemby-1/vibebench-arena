# Proof Matrix

This matrix maps VibeBench claims and capabilities to concrete commands and artifacts. It is intended for reviewers who want to trace a statement to evidence without treating that evidence as more than it is.

| Claim or capability | How it is checked | Command | Primary evidence artifact | What the evidence supports | What it does not prove |
| --- | --- | --- | --- | --- | --- |
| Project configuration validity | Loads and validates `.vibebench/config.yaml` and reports consistency diagnostics. | `python3 -m vibebench config --check --json` | JSON stdout; `config-check.json` / `config-check.md` when generated by `ci` | Config can be parsed and evaluated by VibeBench. | The chosen policy is optimal for the project. |
| Configured command execution | Runs configured local checks and records results. | `python3 -m vibebench check` | `.vibebench/runs/<timestamp>/metrics.json`; `check.log` | Configured commands ran and produced captured status/output. | The checks cover all important behavior. |
| Gate results | Evaluates a run against score, risk, status, and finding thresholds. | `python3 -m vibebench gate --write-gate-summary` | `gate-summary.md`; gate status in CI JSON | The run passed or failed configured gate thresholds. | Passing the gate guarantees code quality. |
| Workflow readiness | Inspects GitHub Actions workflow files for VibeBench readiness signals. | `python3 -m vibebench workflow-check --json` | JSON stdout; `workflow-check.json` / `workflow-check.md` when generated | The workflow files contain detectable VibeBench CI shape and findings. | GitHub Actions or external services will behave correctly. |
| Required CI modes | Requires detected VibeBench CI modes such as `adoption-policy`. | `python3 -m vibebench workflow-check --require-ci-mode adoption-policy --json` | JSON stdout; `workflow-check.json` / `workflow-check.md` in CI policy runs | Expected workflow mode is present or missing. | Future workflow edits preserve the mode. |
| Preflight readiness | Combines read-only setup, onboarding, workflow-template preview, and workflow-check signals. | `python3 -m vibebench preflight --json` | JSON stdout; `preflight.json` / `preflight.md` when generated by `ci --preflight` | The repository has inspectable setup/adoption readiness signals. | The project is ready for production use. |
| Adoption readiness | Reports whether the repo is ready for the requested adoption workflow. | `python3 -m vibebench adoption-ready --json` | JSON stdout | Config, workflow mode, preflight availability, release-check availability, and strict doctor availability are visible. | Adoption will succeed or replace team judgment. |
| Release readiness | Runs local pre-release readiness checks. | `python3 -m vibebench release-check --json` | JSON stdout; `release-check.json` / `release-check.md` when generated | Local release-readiness evidence can be inspected. | A release is approved, safe, or published. |
| Artifact inventory | Lists known artifacts for a run and their availability. | `python3 -m vibebench artifacts --json` | JSON stdout | Reviewers can locate generated run files. | Missing optional artifacts are always a failure. |
| Manifest consistency | Writes or checks the run manifest against the run directory. | `python3 -m vibebench manifest --check` | `manifest.json` | The manifest exists and matches expected run artifacts. | Artifact content is semantically correct. |
| Bundle creation | Packages standard run artifacts into a portable zip. | `python3 -m vibebench bundle` | `vibebench-bundle.zip` | Evidence can be handed off or downloaded as a packet. | The packet is safe to share without manual review. |
| Trend evidence | Summarizes recent run movement. | `python3 -m vibebench trend --write-json --write-summary` | `trend.json`; `trend.md` | Recent score/risk movement can be inspected. | The trend predicts future quality. |
| Comparison evidence | Compares two runs or the latest run against a previous run. | `python3 -m vibebench compare --json` | JSON stdout; `compare.json` / `compare.md` when written | Movement between runs is visible. | A regression finding is a benchmark certification. |
| Evidence-room output | Creates a browseable external review package. | `python3 -m vibebench evidence-room --output-dir /tmp/vibebench-evidence-room --zip` | `/tmp/vibebench-evidence-room/index.html`; `evidence-room.zip`; `share-check.json`; `share-check.md` | Reviewers can inspect a packaged evidence room. | The package has been independently audited. |
| Strict doctor readiness | Checks environment and expected run artifacts for stricter review. | `python3 -m vibebench doctor --strict` | Human stdout; JSON stdout with `--json` | Python, Git, config, workflow, latest run, manifest, bundle, and report readiness are visible. | The environment cannot fail later. |
| JSON output suitability for automation | Commands emit parseable JSON when `--json` is used. | `python3 -m vibebench ci --dry-run --json` | JSON stdout | Automation can parse the planned pipeline without scraping rich text. | Downstream automation handles every failure mode. |
| Shareability scan | Scans a package before external sharing. | `python3 -m vibebench share-check PATH --json` | JSON stdout; `share-check.json` / `share-check.md` in evidence rooms | A local pre-sharing scan found or did not find configured unsafe markers. | The artifact is certified safe to publish. |

For broader context, see [technical due diligence](technical-due-diligence.md), [artifact gallery](artifact-gallery.md), and the [Trust Center](trust-center.md).

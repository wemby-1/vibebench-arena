Trust Center scope

This Trust Center is project-maintained documentation, not a third-party audit or compliance certification. It describes current project behavior and reviewer checks without making unsupported trust claims.

Evidence and reproducibility posture

Local-first

Core evaluation runs from a repository checkout with local files and commands.

No required cloud service

Local proof, site preview, evidence-room, scorecard, and verification commands do not require a hosted VibeBench service.

GitHub Actions artifacts

CI can upload downloadable proof, site preview, and evidence-room artifacts without enabling GitHub Pages automatically.

Generated artifacts

Evidence-room

Self-opening package with local HTML, Markdown, JSON, proof packet, static site preview, Trust Center, and reviewer scorecard files.

Proof packet

Includes proof HTML, JSON, Markdown, manifest, and optional zip output for review.

Static site preview

Packages the public docs entry and setup guide for local inspection.

Reviewer scorecard

Provides neutral checklist files with not-reviewed placeholders rather than external approval claims.

JSON output purity

Commands using --json are expected to write parseable JSON to stdout.

Static HTML safety rules

Generated static HTML avoids scripts, remote URLs, external assets, images, and absolute local paths.

Security and privacy boundaries

Path/privacy hygiene

Shareable artifacts prefer relative paths and placeholders. Review artifacts before sharing outside a local environment.

Security boundary

VibeBench is not a sandbox, hosted security product, or replacement for human security review.

Responsible disclosure

Use the repository security policy for vulnerability reporting expectations.

What the project does not claim

  • No third-party audit.
  • No compliance certification.
  • No fake security certification.
  • No invented customer or adoption claims.
  • No promise of funding or business outcomes.
  • No guarantee that generated code is correct or safe.
  • No automatic publishing, deployment, release, or repository settings change.

Recommended reviewer verification commands

python3 -m vibebench evidence-room --output-dir PATH --zip
python3 -m vibebench evidence-room --verify PATH
python3 -m vibebench proof --verify PATH/proof-packet
python3 -m vibebench site-preview --verify PATH/site-preview
python3 -m vibebench site-check
python3 -m vibebench ci --dry-run --json
python3 -m vibebench release-check
python3 -m vibebench doctor --strict

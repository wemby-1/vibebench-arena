# VibeBench Arena Public Roadmap

This public roadmap describes product direction, not promises. Items are planned or experimental until they are shipped, documented, and verified in the repository.

## Now

VibeBench Arena currently focuses on a local-first quality console for Codex-first / vibe-coding work:

- local quality checks and CI-style dry-run planning
- one-command demo for the checked-in showcase artifacts
- artifact gallery and sample evidence pack
- Git diff and risk review for generated changes
- Markdown, JSON, report, manifest, comparison, badge, and bundle artifacts
- release audit and publish readiness flows that stay local
- GitHub-readable summaries and annotations

## Next

Near-term work is planned around making the product easier to evaluate and more useful for realistic AI coding workflows:

- better sample projects that show clean and risky AI-generated changes
- richer diff and risk explanations for reviewers
- improved HTML report and export experiences
- GitHub PR workflow polish for maintainers and contributors
- more realistic AI coding task packs for demos and repeatable evaluation
- clearer docs for local-first quality gates and artifact review

These items should remain small enough to verify with local commands and repository artifacts.

## Later

Longer-term directions are experimental and may change based on real use:

- team dashboard for historical quality trends and policy visibility
- hosted artifact viewer for teams that want browser-based review
- plugin system for project-specific checks and AI coding workflows
- benchmark-compatible task datasets for reproducible AI-assisted development studies
- enterprise audit workflows for release governance and compliance review
- policy console for team-level risk rules and release readiness expectations

These directions are opportunities, not claims of current availability.

## How Contributors Can Help

Helpful contributions include:

- real AI coding use cases with clear review pain
- demo feedback about what is confusing or missing
- sample projects that produce meaningful artifacts without secrets or private data
- small documentation improvements that make the product thesis easier to understand
- focused PRs that improve report readability, risk explanations, GitHub output, or release-readiness artifacts

Before proposing a large feature, open an issue with the workflow, expected artifact output, and verification commands. VibeBench works best when contributors keep changes local-first, auditable, and easy to reproduce.

## Evaluation Standard

Evaluate VibeBench by what the repository can show today:

- commands that run locally
- artifacts that can be inspected
- docs that explain current and planned scope
- release and publish checks that do not perform hidden side effects
- honest limitations instead of inflated traction claims
